# 確定申告ツール MVP 要件定義（v1.0）

最終更新: 2026-03-05

## 1. 目的
- 白色申告向けに、レシート集計入力を高速化する。
- 手作業の月別・カテゴリ別転記を置き換え、月次/年次の収支を即時確認できるようにする。

## 2. 対象範囲
- 対応OS: Windows
- 形態: デスクトップGUI（Tauri + React）
- 保存先: ローカルSQLite

## 3. 機能要件（MVP）
1. プロファイル管理
- 初期2名運用を想定しつつ、追加/編集可能。
- プロファイル切替時、入力・一覧・集計を全面切替。

2. 明細入力
- 入力項目: 年（西暦4桁）、月（1-12）、カテゴリ、金額（税込）。
- 入力方式: 年/月/カテゴリを先に選択し、金額をEnterで連続登録。
- 入力後の修正・削除が可能。
- 前回選択（年/月/カテゴリ/プロファイル）を保持。

3. カテゴリ管理
- カテゴリの追加・編集が可能。
- カテゴリ削除時は2モードを提供:
  - 強制削除（紐づく明細も削除）
  - 別カテゴリへ移管して削除

4. 集計
- 月次（指定年/月）: カテゴリ別合計、収入合計、支出合計、収支（収入-支出）。
- 年次（指定年）: カテゴリ別合計、収入合計、支出合計、収支。

5. Excel取込
- 対象シート: `データ` のみ。
- 列対応: 人物/年/月/カテゴリ/実績。
- 変換:
  - `駐車場代` -> `交通費`
  - `収入` と `総売上` は除外
  - `売上` は取り込む（旧データ保全）
- `売上` は旧データ表示専用カテゴリとし、新規入力では選択不可。

## 4. 初期カテゴリ
- 収入（新規入力可）: `本業売上`, `出前売上`
- 収入（表示専用）: `売上`
- 支出: `固定資産`, `家賃`, `租税公課`, `光熱費`, `交通費`, `通信費`, `接待費`, `保険料`, `修繕費`, `消耗品`, `車両代`, `雑費`

## 5. 非機能要件
- キーボード中心操作（Tab遷移 + Enter登録）。
- 金額は整数円で扱い、浮動小数点を使わない。
- プロファイル混在を防ぐため、全クエリで `profile_id` を必須条件とする。
- カテゴリ削除/移管はトランザクションで整合性を保証。

## 6. 受け入れ条件（AC）
1. プロファイル切替で、表示データと集計が完全に切り替わる。
2. 年/月/カテゴリを固定した状態で、金額のEnter連続入力ができる。
3. 明細の修正/削除後、一覧と集計が即時に一致更新される。
4. 月次・年次で、カテゴリ別合計と収支が正しく計算される。
5. Excel取込後、`駐車場代` が `交通費` に正規化され、`収入`/`総売上` は除外される。
6. `売上` は参照表示できるが、新規入力フォームでは選択できない。

## 7. エッジケース境界（EC）
1. 年は2000-2100、月は1-12以外を拒否。
2. 金額は正の整数円のみ受理。
3. 同名カテゴリ追加を拒否。
4. カテゴリ移管で同一カテゴリ指定を拒否。
5. 無効行（列不足/不正値）はスキップして件数を結果表示。

## 8. MVPスコープ外
- e-Tax提出連携
- OCR/レシート画像解析
- クラウド同期
- 申告書自動生成
