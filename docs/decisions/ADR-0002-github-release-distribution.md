# ADR-0002: GitHub ReleasesでWindows配布を標準化する

- 日付: 2026-03-05
- ステータス: Accepted

## 背景

- 本プロジェクトはWindows向けデスクトップアプリ（Tauri）である。
- OSS公開時に、利用者がソースビルドなしで導入できる配布経路が必要。
- 手作業ビルドのみでは再現性と運用負荷に課題がある。

## 決定

- 配布の正規経路を GitHub Releases とする。
- `v*` タグ push をトリガとして、GitHub Actions で Windows バンドルを自動生成する。
- 生成対象は以下の2形式とする。
  - NSIS `.exe`
  - MSI `.msi`

## 理由

- 利用者は Releases から直接ダウンロードできる。
- タグ駆動により、配布物とバージョンの対応が明確になる。
- ビルドの再現性をCIで担保できる。

## 影響

- リリース時はバージョン更新後に `v*` タグ運用を行う。
- GitHub Actions の権限設定（workflow write）が必要。

## 代替案

- ローカル手動ビルド配布:
  - メリット: 初期設定が少ない
  - デメリット: 作業者依存、再現性低下、配布漏れリスク
