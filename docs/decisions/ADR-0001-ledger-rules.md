# ADR-0001: 集計/取込/入力ルールの固定

- 日付: 2026-03-05
- ステータス: Accepted

## Context
- 白色申告MVPでは、集計誤差と二重計上が最重要リスク。
- 既存Excelには `売上/本業売上/出前売上/収入/総売上/駐車場代` が混在。

## Decision
1. 金額は整数円のみ（小数禁止）。
2. 収支計算は `category.kind` に基づいて決定論的に計算。
3. importルール:
   - `駐車場代 -> 交通費`
   - `収入` と `総売上` は除外
   - `売上` は取り込み対象（旧データ保全）
4. カテゴリの `input_enabled` はユーザー設定で制御する（カテゴリ名で固定しない）。

## Consequences
- 二重計上と計算揺れを抑制できる。
- 旧データ互換性を維持しつつ、カテゴリ運用をユーザーごとに柔軟化できる。
- 一方で、DB直接編集の完全防止は未実装のため、将来DB制約強化が望ましい。
