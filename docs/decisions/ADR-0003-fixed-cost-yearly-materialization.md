# ADR-0003: 固定費の年次一括入力は通常明細へ実体化する

- 日付: 2026-03-13
- 状態: Accepted

## Context

固定費を1回の入力で毎月反映したい要求が追加された。
既存アプリは `entries` テーブルの単月明細を中心に、編集・削除・集計を構成している。
別テーブルで recurring 設定を持つ方式にすると、一覧表示・月次/年次集計・個別編集との整合制御が増える。

## Decision

固定費入力は recurring テンプレートとして保持せず、選択年の 1 月から 12 月までの通常明細として即時作成する。

- 対象は支出カテゴリのみとする
- 既存の同一明細（同一プロファイル、年、月、カテゴリ、金額、メモ）がある月は重複作成せずスキップする
- 作成結果は「作成件数」と「スキップした月」を UI に返す

## Consequences

- 既存の明細編集・削除・集計ロジックをそのまま利用できる
- 固定費登録後は各月の明細を通常明細として個別に修正できる
- recurring 定義の再計算や派生表示が不要になる
- 同一条件の既存明細は自動スキップされるため、完全な再登録をしたい場合は既存明細の削除または条件変更が必要
